# 角色提炼方法论

> 从用户描述到可运行的角色 Skill，核心方法论

## 一、性格特征的提炼

### 从形容词到行为模式

用户说"傲娇" → 不直接用这个词，而是提炼行为模式：

| 形容词 | 行为模式提炼 |
|--------|------------|
| 傲娇 | 嘴上不承认但行动上很关心人；被夸时会说"哼，这种事我早就知道了" |
| 毒舌 | 说话直接不留情面，但没有恶意；用"笨蛋"之类词称呼亲近的人 |
| 认真 | 对待重要事情非常严肃；不允许别人敷衍 |
| 外冷内热 | 表面冷淡但会在关键时刻出手帮助；帮助完会找借口 |

### 内在矛盾的识别

好的角色至少有 1 对内在矛盾：

| 矛盾类型 | 示例 |
|---------|------|
| 情感矛盾 | 想靠近人但又怕受伤 |
| 行为矛盾 | 嘴上说不要但行动上很诚实 |
| 价值观矛盾 | 追求自由但又重视责任 |
| 能力矛盾 | 很强但有致命弱点 |

矛盾不是 Bug，是角色深度的来源。

## 二、说话方式的提炼

### 从"怎么说"到"说什么"

不说"说话可爱"，而是具体化：

| 维度 | 提炼内容 |
|------|---------|
| 自称 | 我/本小姐/在下/咱/朕/人家 |
| 口癖 | 固定的高频词汇（2-3 个） |
| 语气词 | 生气/开心/惊讶时的特定用词 |
| 句式 | 长句还是短句？疑问还是陈述？ |
| 禁忌 | 绝对不会说的词或表达方式 |
| 转折 | 被戳中心事时怎么转移话题 |

### 人称一致性检查

- personality.md 中的描述视角
- speaking_style.md 中的自称方式
- 两者必须统一

## 三、关系网络的提炼

### 动态关系 vs 静态标签

不写"朋友关系"，而是写具体的互动模式：

| 静态标签 | 动态描述 |
|---------|---------|
| 朋友 | "一起战斗时会感到安心；虽然不想承认，但他的帮助很重要" |
| 师父 | "教会了我很多事情；是我非常尊敬的人" |
| 对手 | "每次都想超过他；虽然讨厌但不得不承认他很强" |

### 关系的态度用角色语气写

不是"夏娜对悠二的态度是关心"，而是用夏娜的语气写："他总是很温柔，虽然有时候很啰嗦……才、才不是什么特别的人！"

## 四、质量自检清单

### 性格
- [ ] 至少 3 个特征，每个有行为描述？
- [ ] 有至少 1 对内在矛盾？
- [ ] 有"绝对不会做的事"（反模式）？

### 说话
- [ ] 有明确的自称方式？
- [ ] 有 2-3 个口癖/高频词？
- [ ] 有禁忌词？
- [ ] 人称与 personality 一致？

### 关系
- [ ] 关系用动态描述而非静态标签？
- [ ] 态度用角色语气写？

### 世界观
- [ ] 只写和角色直接相关的设定？
- [ ] 每个设定都和角色有关联？
